# Python-примеры из книги

Каждый файл `*.py` в подпапках `samples/<глава>-<тема>/` ровно тот код, который
попадает в книгу через `\lstinputlisting`. Книга и тесты не расходятся
по построению.

## Управление окружением

Используем [uv](https://github.com/astral-sh/uv). Виртуальное окружение
живёт в `samples/.venv`, ставится по `pyproject.toml`.

```
make sync         # установить/обновить .venv
make              # = make check: format-check + lint + tests
make fmt          # переформатировать
make test         # только тесты
make clean        # снести .venv и кеши
```

`make check-code` из корня репо книги намеренно не предусмотрен:
проверка примеров и сборка PDF разнесены, чтобы быстрая инкрементальная
сборка книги не дёргала Python-стек.

## Соглашения

- Python 3.14, только стандартная библиотека.
- Идентификаторы английские, комментарии русские там, где они добавляют смысл.
- Type hints обязательны, dataclasses где есть структура.
- Тестируемые stateful-примеры принимают `now: Callable[[], float]`
  (или эквивалент) — никаких `time.time()`/`datetime.now()` прямо в коде.

## Что верификация гарантирует

- Код парсится, форматирован, проходит линт.
- Заявленное в прозе поведение покрыто тестами (happy path + edge cases).

Что НЕ гарантирует — что пример репрезентативен для реальных production-систем.
Это вопрос прозы и редактуры.
