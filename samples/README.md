# Python-примеры из книги

Каждый файл `*.py` в подпапках `samples/<глава>-<тема>/` — ровно тот код,
который попадает в книгу через `\codefile`. Книга и тесты не расходятся
по построению.

## Окружение

Используем [uv](https://github.com/astral-sh/uv). Виртуальное окружение
живёт в `samples/.venv`, ставится по `pyproject.toml`.

```
make sync         # установить/обновить .venv
make              # = make check: ruff format --check + ruff check + pytest
make fmt          # переформатировать
make test         # только тесты
make clean        # снести .venv и кеши
```

`make check-code` из корня репо намеренно не предусмотрен: проверка
примеров и сборка PDF разнесены, чтобы быстрая инкрементальная сборка
книги не дёргала Python-стек.

## VSCode

Открыть репо в VSCode, согласиться с расширениями из `.vscode/extensions.json`
(ms-python, Pylance, ruff, editorconfig). Test Explorer автоматически
подхватит `samples/.venv` и покажет все тесты, сгруппированные по
sample-каталогам; формат при сохранении совпадает с `make fmt`.

При добавлении нового sample-каталога нужно добавить путь
в `python.analysis.extraPaths` в `.vscode/settings.json` (одна строка),
иначе Pylance не резолвит `from <module> import …` в редакторе. Pytest
эту проблему обходит сам.

## Соглашения

- Python 3.14, только стандартная библиотека.
- Идентификаторы английские, комментарии русские там, где они добавляют смысл.
- 2-пробельный отступ (`.editorconfig` + `indent-width = 2` в ruff).
- Type hints обязательны, dataclasses где есть структура.
- Stateful-примеры принимают `now: Callable[[], float]` или эквивалент —
  никаких `time.time()`/`datetime.now()` прямо в коде.
- Никаких module-level docstrings: пояснения к примеру живут в прозе книги.
  Method/class docstrings — по необходимости.
- Em-dash (`—`) в комментариях и строках не использовать: `listings`
  не понимает многобайтовые UTF-8 пунктуационные знаки даже с `listingsutf8`.
  Заменять на `--`. Кириллица в идентификаторах не используется,
  в комментариях работает через `literate=`-карту в `src/styles.tex`.

## Что верификация гарантирует

- Код парсится, форматирован, проходит линт.
- Заявленное в прозе поведение покрыто тестами (happy path + edge cases).

Что НЕ гарантирует — что пример репрезентативен для реальных
production-систем. Это вопрос прозы и редактуры.
