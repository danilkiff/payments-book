\begin{kaobox}[title=Что вы узнаете из этой главы]
Система быстрых платежей~--- не~ещё~одна карточная система, а~принципиально
иная архитектура: без~эмитента и~эквайера в~классическом смысле,

мобильного эквайринга SBPay, логику подписок и~акцепта, принципиальную
финальность (irrevocability) переводов, тарифы и~лимиты, а~также
отличие СБП от~цифрового рубля.
\end{kaobox}


\section{Архитектура: трёхуровневая схема}
\label{sec:sbp-architecture}

Чтобы понять СБП, удобнее всего начать с~того, чего в~ней~\emph{нет}.
Нет эмитента, выпустившего карту и~установившего кредитный лимит.
Нет эквайера, подписавшего договор с~мерчантом и~несущего ответственность
за~его~транзакции. Нет межбанковской комиссии (interchange), которую
эквайер платит эмитенту. Есть только три участника: Банк России
как~расчётный центр и~регулятор, НСПК как~операционный центр,
обеспечивающий маршрутизацию и~контроль сообщений, и~банки-участники~---
платёжные агенты своих клиентов~\todocite{Правила платёжной системы
СБП НСПК nspk.ru 2024}.

Мгновенный перевод становится возможным потому, что~Банк России
переводит деньги между корреспондентскими счетами банков-участников
в~режиме реального времени через систему БЭСП (Банковские электронные
срочные платежи) или~её~преемника~--- сервисы системы ПСБР (платёжная
система Банка России). Для~конечного пользователя это~означает, что~деньги
поступают на~счёт получателя почти мгновенно~--- в~отличие от~карточного
мира, где~settlement происходит на~следующий рабочий день.

Важно разграничить роли: НСПК в~СБП~--- это~операционный, а~не~расчётный
центр. Деньги через НСПК не~проходят; НСПК только маршрутизирует
сообщения, проверяет их~форматы и~обеспечивает адресацию. Расчёты
идут через Банк России. Это~фундаментальное отличие от~карточной
модели, где~\scheme{Visa} или~\scheme{Mastercard} является и~операционным
центром, и~участником расчётной цепочки.


\section{Сценарии платежей}
\label{sec:sbp-scenarios}

СБП поддерживает несколько принципиально разных сценариев, каждый
из~которых имеет собственный flow и~технические особенности~\todocite{Правила
платёжной системы СБП НСПК nspk.ru 2024}.

\textbf{C2C (гражданин~— гражданину):} перевод по~номеру мобильного
телефона. Отправитель вводит номер телефона получателя в~приложении
своего банка; НСПК по~реестру привязок определяет банк и~счёт получателя.
Привязка номера телефона к~счёту~--- обязательная процедура, которую
получатель должен выполнить в~своём банке заранее. Если~номер
привязан к~нескольким банкам, отправитель может выбрать целевой банк.

\textbf{C2B статический QR (гражданин~— бизнесу):} QR-код привязан
к~мерчанту и~его~реквизитам, но~не~содержит сумму. Покупатель сканирует
код, вводит сумму вручную в~приложении своего банка и~подтверждает
платёж. Такой код можно напечатать один раз и~использовать многократно.
Недостаток~--- покупатель сам вводит сумму, что~создаёт риск ошибки
и~дополнительный шаг в~UX~\todocite{Правила платёжной системы СБП НСПК
nspk.ru 2024}.

\textbf{C2B динамический QR:} QR-код генерируется на~конкретный заказ
и~содержит заранее заполненную сумму. Покупатель сканирует код, видит
точную сумму и~только подтверждает платёж. Это~более безопасная и~конвертирующая
модель: меньше шагов, нет риска ошибочной суммы, код одноразовый
(привязан к~конкретной транзакции). Большинство e-commerce интеграций
используют именно динамический QR~\todocite{Правила платёжной системы
СБП НСПК nspk.ru 2024}.

\textbf{B2C (бизнес~— гражданину):} выплаты клиентам~--- возвраты,
кешбэк, выигрыши, выплаты гиг-работникам. Инициатор~--- юридическое
лицо; получатель~--- физическое лицо по~номеру телефона или~реквизитам счёта.

\textbf{B2B (бизнес~— бизнесу):} корпоративные переводы, уплата
налогов, расчёты с~контрагентами. До~появления СБП большинство таких
платежей шли через платёжные поручения с~однодневным или~двухдневным
сроком исполнения; СБП обеспечивает мгновенное межбанковское зачисление.

\section{SBPay: мобильный эквайринг}
\label{sec:sbp-sbpay}

SBPay~--- сценарий C2B, оптимизированный для~мобильной торговли.
Вместо того чтобы показывать пользователю QR-код (который нужно
сканировать камерой), приложение мерчанта формирует deep link~--- специальную
ссылку, которая открывает банковское приложение пользователя напрямую,
уже~с~заполненными реквизитами платежа~\todocite{Правила платёжной системы
СБП НСПК nspk.ru 2024}.

Механизм deep link работает следующим образом. Пользователь нажимает
кнопку \enquote{Оплатить через СБП} в~приложении мерчанта. Приложение
формирует URI вида \texttt{bank://sbp?...} или~использует схему
\texttt{https://qr.nspk.ru/...} с~параметрами транзакции. Операционная
система перехватывает этот URI и~предлагает открыть одно из~установленных
банковских приложений, поддерживающих СБП. Пользователь выбирает
свой банк (или~пропускает выбор, если~он~уже задан по~умолчанию),
видит детали платежа и~подтверждает его~биометрией или~PIN-кодом.

Принципиальное отличие от~NFC-платежей: SBPay~--- программный протокол
поверх стандартных механизмов мобильной ОС (deep links и~inter-app
communication), тогда как~NFC~--- аппаратный протокол с~tap-and-pay
в~одно касание. NFC не~требует разблокировки устройства и~открытия
приложения; SBPay требует активного участия пользователя. Зато SBPay
не~требует NFC-терминала: мерчанту достаточно показать QR или~создать
deep link.

\begin{practice}
Для~e-commerce наиболее конвертирующий сценарий СБП~--- динамический
QR на~экране, а~не~статический, по~двум причинам. Во-первых, покупателю
не~нужно вводить сумму вручную: меньше шагов~--- выше конверсия.
Во-вторых, динамический QR одноразовый: повторное сканирование не~создаёт
дубль платежа. На~мобильных устройствах эффективнее предлагать
SBPay (deep link) вместо QR: пользователю не~нужно использовать камеру
для~сканирования собственного экрана~--- это~неудобный UX.
\end{practice}


\section{Подписки через СБП}
\label{sec:sbp-subscriptions}

СБП изначально не~был предназначен для~рекуррентных платежей: каждый
перевод~--- явное действие пользователя. Однако в~2022--2023~годах
НСПК ввела механизм акцепта (consent), позволяющий реализовать автоматическое
списание~\todocite{Правила платёжной системы СБП НСПК nspk.ru 2024}.

Механизм работает следующим образом. Пользователь однократно даёт
разрешение (акцепт) на~автосписание в~пользу конкретного мерчанта.
Акцепт содержит параметры: максимальную сумму одного списания,
максимальную суммарную сумму за~период, периодичность и~срок действия
разрешения. После оформления акцепта мерчант вправе инициировать
списание в~рамках этих параметров~--- без~дополнительного подтверждения
со~стороны пользователя в~каждом случае.

Принципиальное отличие от~stored credential в~карточном мире (которое
мы~подробнее рассмотрим в~главе~\ref{ch:subscriptions}): у~мерчанта
нет карточных данных, которые можно \enquote{сохранить}. Акцепт СБП~---
это~разрешение, зарегистрированное в~системе НСПК, а~не~реквизиты,
хранимые на~стороне мерчанта. Отзыв акцепта пользователь выполняет
в~приложении своего банка или~непосредственно в~кабинете мерчанта.


\section{Финальность и отсутствие chargeback}
\label{sec:sbp-finality}

Перевод в~СБП является финальным~--- и~это~не~просто технический
термин, а~юридическое состояние: завершённый перевод не~может быть
отозван в~одностороннем порядке ни~банком-отправителем, ни~НСПК,
а~деньги, поступившие получателю, принадлежат ему~безвозвратно~\sidecite{fz-161}.

Из~финальности прямо вытекает отсутствие chargeback~--- механизма,
знакомого из~карточного мира. В~карточных схемах chargeback позволяет
держателю карты оспорить транзакцию у~своего банка-эмитента, который
при~наличии оснований инициирует возврат через платёжную сеть. В~СБП
такой четырёхсторонней модели~---~эмитент, платёжная сеть, эквайер,~---
попросту нет. Договор заключён между пользователем и~его~банком;
правила схемы не~создают механизма принудительного возврата для~получателя.

\begin{important}
Финальность СБП~--- двусторонний нож. Для~мерчанта это~хорошо:
нет friendly fraud (ситуации, когда добросовестный покупатель оспаривает
реальную транзакцию ради возврата денег)~--- явления, которое
в~карточном мире обходится отрасли в~миллиарды долларов ежегодно.
Но~операционная ошибка~--- неверная сумма, ошибочный платёж~--- не~откатывается
автоматически. Возврат возможен только как~новая транзакция: мерчант
по~своей воле переводит деньги обратно. Если мерчант отказывается,
пользователь защищён только претензионным порядком по~\law{161-ФЗ}
(обращение в~банк, затем в~суд)~--- но~не~автоматическим chargeback.
\end{important}

Что~делать при~фроде? Если~пользователь перевёл деньги мошеннику
(social engineering), он~должен немедленно обратиться в~свой банк.
Банк вправе направить запрос в~НСПК на~заморозку средств у~банка-получателя
в~рамках механизмов противодействия мошенничеству, предусмотренных
правилами СБП. Это~возможно только в~узком временно́м окне до~того,
как~мошенник вывел деньги~\sidecite{fz-161}.

\section{Тарифы, лимиты, обязательное подключение}
\label{sec:sbp-tariffs}

Тарифы СБП устанавливаются НСПК и~Банком России и~существенно ниже
карточного interchange~\todocite{актуальные тарифы платёжной системы
СБП на~сайте НСПК nspk.ru 2024}.

Для~физических лиц переводы C2C бесплатны в~пределах 100~тыс.~рублей
в~месяц; сверх этой суммы~--- 0,5\% от~суммы перевода, но~не~более
1\,500~рублей за~операцию. Лимит на~один перевод C2C~--- 1~млн~рублей.

Для~мерчантов (C2B) комиссия дифференцирована по~категориям:
от~0,4\% для~отдельных социально значимых категорий (фармацевтика,
медицина, транспорт) до~0,7\% для~большинства других. Это~существенно
ниже ставок карточного эквайринга, что~делает СБП привлекательным
для~мерчантов с~высокими оборотами.

Обязательное подключение: банки, имеющие банковскую лицензию
и~обслуживающие более 500~тысяч клиентов-физических лиц, обязаны
участвовать в~СБП~\todocite{актуальный порог по~числу клиентов для~обязательного
участия в~СБП по~состоянию на~2024~год}. На~практике к~СБП подключены
все~системно значимые банки России и~большинство средних банков.


\section{Цифровой рубль: где пересекается с СБП, где нет}
\label{sec:sbp-cbdc}

С~принятием \law{340-ФЗ} в~2023~году цифровой рубль получил
законодательную основу~\sidecite{fz-340}. Важно понять, как~цифровой
рубль соотносится с~СБП~--- потому что~оба продвигаются как~\enquote{цифровые
платежи} и~оба имеют отношение к~НСПК, однако это~принципиально
разные уровни денежной системы.

СБП оперирует обычными (безналичными) деньгами: средствами на~расчётных
счетах в~коммерческих банках. Деньги хранятся у~банков, которые
несут перед клиентами обязательства и~страхуются через систему АСВ.
СБП~--- только канал быстрого перевода этих денег.

Цифровой рубль~--- третья форма денег наряду с~наличными и~безналичными.
Это~токен, эмитированный и~хранящийся непосредственно в~Банке России,
а~не~в~коммерческом банке. Клиент держит цифровые рубли в~кошельке
на~платформе Банка России; банк-посредник только предоставляет
интерфейс доступа, но~не~хранит средства.

Пересечение: НСПК выполняет роль операционного центра как~для~СБП,
так~и~для~цифрового рубля в~части розничных операций. Технически
обе системы используют общую инфраструктуру НСПК для~маршрутизации.

Принципиальные отличия: цифровой рубль поддерживает офлайн-операции
(перевод токенов без~доступа к~сети), умные контракты (программируемые
условия использования средств) и~целевое использование (например,
субсидии, которые можно потратить только в~определённых категориях).
СБП не~работает офлайн и~не~поддерживает умные контракты.
Для~держателей средств есть и~существенное отличие в~гарантиях:
остатки на~счетах в~коммерческом банке застрахованы АСВ до~1,4~млн~рублей;
цифровые рубли хранятся непосредственно у~Банка России~---
государственного эмитента~\sidecite{fz-340}.


\section*{Итоги главы}

\begin{itemize}
  \item СБП~--- трёхуровневая система: Банк России (расчёты),
        НСПК (операции), банки-участники (клиентский интерфейс);
        нет эмитента/эквайера/interchange в~карточном смысле.
  \item Пять сценариев: C2C (телефон), C2B статический QR, C2B
        динамический QR, B2C (выплаты), B2B (корпоративные);
        динамический QR и~SBPay~--- наиболее конвертирующие для~ритейла.
  \item Акцепт (consent) позволяет реализовать автоматическое
        списание~--- аналог stored credential, но~без~хранения
        карточных данных на~стороне мерчанта.
  \item Финальность перевода~--- нет chargeback, нет friendly fraud,
        но~операционная ошибка откатывается только добровольным
        возвратом мерчанта.
  \item СБП vs карты: СБП выигрывает по~стоимости (0,4--0,7\%)
        и~скорости settlement; карты~--- по~международному охвату,
        chargeback и~крупным лимитам.
  \item Обязательное подключение~--- банки с~более 500~тыс.~клиентов;
        лимит C2B~--- 1~млн~₽ за~операцию.
  \item Цифровой рубль~--- третья форма денег (CBDC), не~СБП;
        хранится у~Банка России, поддерживает офлайн и~смарт-контракты.
\end{itemize}
