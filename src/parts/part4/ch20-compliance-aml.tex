\chapter{Комплаенс и AML}
\label{ch:compliance-aml}

\begin{kaobox}[title=Что вы узнаете из этой главы]
  Комплаенс в~платёжном бизнесе~--- не~набор бумаг для~регулятора, а~живая
  операционная функция, встроенная в~каждый шаг жизненного цикла клиента
  и~транзакции. Из~этой главы вы~узнаете, что~требует \law{115-ФЗ} от~платёжного
  оператора; чем~KYC физлиц отличается от~KYB мерчантов; где в~транзакционном
  flow встраивается санкционный скрининг и~как~управлять false positive
  без~потери скорости; почему кейс QIWI~--- урок не~про~один банк; и~как~FATF-рекомендации
  реализованы в~регуляторном контексте СНГ.
\end{kaobox}

Эту главу лучше читать как операционный конвейер, а~не~как
сборник норм. Сначала бизнес пускает клиента или~мерчанта
внутрь, затем проверяет его по~санкционным и~AML-правилам,
потом наблюдает за~поведением после онбординга и,~если риск
реализуется, переводит случай в~режим эскалации и~отчётности.


\section{115-ФЗ и~платёжный бизнес}
\label{sec:compliance-115fz}

\law{115-ФЗ}~--- Федеральный закон \enquote{О~противодействии легализации
  (отмыванию) доходов, полученных преступным путём, и~финансированию терроризма}~---
является основным антиотмывочным законодательством России~\sidecite{fz-115}.
Для~платёжного бизнеса закон устанавливает три группы обязательств.

Первая группа касается идентификации: оператор обязан идентифицировать
клиента до~проведения операции. Закон предусматривает три уровня
идентификации: упрощённая~--- для~электронных кошельков с~ограниченным
функционалом (пороги на~остаток и~оборот); стандартная~--- документальная,
обычно через паспорт или~ЕСИА; полная~--- при~операциях сверх определённых
порогов или~при~высоком риск-профиле. Конкретные пороговые значения
для~упрощённой идентификации регулярно корректируются: на~момент написания
книги~---15\,000~₽ для~единовременного платежа и~40\,000~₽ ежемесячного
оборота~\sidecite{fz-115}\todocite{актуальные пороги упрощённой идентификации по~115-ФЗ с~последними поправками}.

Вторая группа обязательств связана с~мониторингом: оператор обязан выявлять операции,
подпадающие под критерии обязательного контроля~--- например, операции
на~сумму от~600\,000~₽~--- и~направлять сообщения в~Росфинмониторинг.
Кроме того, оператор обязан выявлять необычные операции~--- те, которые
не~подпадают под формальные критерии, но~вызывают подозрение на~основании
поведенческих признаков или~риск-профиля~\sidecite{fz-115}.

Наконец, третья группа~--- заморозка средств: при~наличии клиента или~бенефициара
в~перечне организаций и~физических лиц, причастных к~терроризму и~экстремизму
(перечень Росфинмониторинга), оператор обязан заморозить средства и~сообщить
об~этом в~Росфинмониторинг незамедлительно~\sidecite{fz-115}.

Связь с~лицензионными требованиями прямая: ЦБ~РФ при~выдаче лицензии
НКО или~ОПКЦ проверяет наличие работоспособной AML-системы~--- регламентов,
программного обеспечения, обученного персонала. Плановые и~внеплановые
проверки ЦБ~--- один из~основных рисков для~платёжного оператора
(глава~\ref{ch:regulation}).


Итак, \law{115-ФЗ} задаёт три базовые обязанности: знать клиента,
мониторить операции и~уметь замораживать средства по~формальным
основаниям. Дальше начинается операционная работа: как именно
эти обязанности разворачиваются в~KYC, KYB, скрининг
и~последующий мониторинг.

\section{KYC при~онбординге физлиц}
\label{sec:compliance-kyc}

Know Your Customer~(KYC)~--- это~процесс верификации личности клиента.
Для~платёжного оператора онбординг физлица~--- это~точка входа в~отношения
с~риском, и~именно здесь закладывается риск-профиль, который определяет
дальнейшие ограничения и~мониторинг.

Стандартный flow KYC для~платёжного приложения в~России: регистрация
по~номеру телефона (минимальный уровень~--- анонимный пользователь
с~жёсткими лимитами); подтверждение через ЕСИА (Госуслуги)~--- упрощённая
идентификация, позволяет пополнять кошелёк до~установленного порога;
загрузка паспорта и~селфи или~проверка через видеоверификацию~---
стандартная идентификация, снятие большинства ограничений; полная
идентификация с~личным визитом или~нотариальным заверением~--- для~операций
с~максимальными суммами.

Проверки при~идентификации: верификация паспорта (ФМС, МВД через API
или~BI.ZONE); проверка по~перечню Росфинмониторинга~--- организации
и~физлица, причастные к~терроризму; проверка по~перечню ЦБ (недобросовестные
финансовые организации, блокировки по~161-ФЗ); проверка по~санкционным
спискам~--- OFAC, ЕС, ООН; оценка по~ПЭП-спискам (политически значимые
лица)~--- для~них усиленная идентификация обязательна.

\begin{termdef}{Политически значимое лицо (ПЭП)}
  Физическое лицо, которое занимает или~занимало значительные публичные
  должности: глава государства, депутат, судья, высокопоставленный чиновник,
  руководитель госпредприятия, их~близкие родственники и~известные
  соратники. Для~ПЭП применяется Enhanced Due Diligence (EDD)~---
  расширенная идентификация и~усиленный мониторинг операций.
\end{termdef}


\section{KYB при~онбординге мерчантов}
\label{sec:compliance-kyb}

Know Your Business~(KYB)~--- верификация юридического лица~---
значительно сложнее KYC физлица. Мерчант несёт операционный риск (может
накапливать возвраты и~chargeback), кредитный риск (получает деньги
за~товары, которые ещё~не~поставил)~и~регуляторный риск (может быть
использован для~отмывания денег). Андеррайтинг мерчанта~--- это~оценка
всех трёх рисков одновременно.

Стандартный пакет документов при~KYB: выписка ЕГРЮЛ/ЕГРИП (автоматически
через ФНС API), паспортные данные директора и~учредителей, паспорт
бенефициара, данные о~структуре собственности (для~выявления UBO~---
ultimate beneficial owner), банковские реквизиты для~settlement, описание
бизнеса и~образцы сайта или~приложения.

Проверки по~MCC имеют особое значение: некоторые категории мерчантов
несут повышенный риск по~умолчанию~--- gambling (MCC 7993/7994/7995),
adult content, nutraceuticals, travel. Эквайеры устанавливают специальные
условия и~возможные ограничения для~этих категорий~\sidecite{visa-core-rules}.
Подмена MCC~--- искусственное отнесение транзакций к~более~\enquote{мягкой}
категории~--- нарушение правил схем и~повод для~штрафов или~расторжения
договора.

PayFac-модель переносит KYB на~PayFac-провайдера, но~не~снимает
ответственность полностью. Перед схемой PayFac всё равно отвечает
за~поведение своих sub-мерчантов.

Структуру ответственности и~лицензирование PayFac рассматриваем
в~главе~\ref{ch:issuer-acquirer}.

\section{Санкционный скрининг}
\label{sec:compliance-sanctions}

Санкционный скрининг~--- проверка клиента, бенефициара и~контрагента
по~санкционным спискам~--- обязателен для~любого платёжного оператора,
работающего с~международными расчётами.

Основные списки:
\begin{itemize}
  \item OFAC~SDN (Specially Designated Nationals and Blocked Persons)~---
        список Министерства финансов США;
  \item Consolidated List~ЕС;
  \item санкционные списки ООН;
  \item для~российских операторов~--- собственные перечни ЦБ~РФ.
\end{itemize}

Первый практический вопрос~--- где именно встраивать скрининг. Трёх точек обычно недостаточно;
нужны как~минимум четыре. При~онбординге~--- до~активации аккаунта:
это~жёсткий блок, нарушение которого~--- прямая санкционная ответственность.
При~каждой транзакции в~реальном времени~--- особенно критично для~трансграничных
переводов: данные о~получателе появляются только в~момент перевода.
При~обновлении данных клиента~--- смена имени, реквизитов. При~обновлении
санкционных списков~--- ретроспективный прогон: новое лицо попало в~список,
ваши существующие клиенты должны быть проверены.

Real-time скрининг при~каждой транзакции создаёт проблему latency.
Если проверка занимает 200~мс, это уже заметный вклад в~общее время
авторизации.

Промышленные решения~--- Accuity, Dow~Jones, Comply~Advantage,
LexisNexis~--- обычно работают на~in-memory базах и~дают ответ
за~5--20~мс при~нормальной нагрузке~\todocite{данные о~latency коммерческих санкционных скрининговых систем}.

Третий операционный вызов~--- управление ложными совпадениями (false positive management). Санкционные списки содержат распространённые
имена~--- Иванов, Mohammed~Al-Rashid~--- и~fuzzy matching даёт большое
число ложных совпадений. Управление false positive~---
это~процесс с~эскалационным workflow: автоматическая система помечает
транзакцию как~\enquote{требует проверки}; операционный сотрудник
изучает контекст (гражданство, адрес, история операций)~--- большинство
помеченных транзакций оказываются легитимными однофамильцами; подтверждённое
совпадение~--- заморозка и~сообщение регулятору. Ключевая метрика качества~---
соотношение автоматически разрешённых к~эскалированным: высокий процент
эскалации означает либо плохо настроенный matching-порог, либо перегруженную
операционную команду.


\section{Мониторинг мерчантов после онбординга}
\label{sec:compliance-merchant-monitoring}

Онбординг~--- одноразовая проверка; мошенники это~знают. Bust-out
мошенничество и~отмывание через мерчантов работают именно потому,
что~после онбординга мерчант нередко выпадает из~зоны активного
внимания. Системный поведенческий мониторинг после онбординга~---
не~опция, а~регуляторное требование и~коммерческая необходимость.

Поведенческие сигналы, требующие расследования: резкий рост объёма
транзакций (в~2~и~более раз за~короткий период) без~очевидного коммерческого
объяснения; изменение паттерна~--- мерчант, работавший с~чеком~500~₽,
вдруг проводит транзакции по~50\,000~₽; высокий процент возвратов или~chargeback
(выше 1\%~― сигнал для~диспут-программ, выше~5\%~--- сигнал AML);
MCC mismatch~--- транзакции не~соответствуют заявленному виду деятельности;
нетипичная география клиентов~--- российский ресторан с~80\%~иностранных
карт.

Мониторинг реализуется через transaction monitoring rules~--- по~сути
velocity checks для~мерчантского профиля. В~промышленных системах
это слой поверх авторизационной базы с~агрегатами по~мерчанту:
скользящее среднее, стандартное отклонение, выброс-детекция.
Результат~--- очередь кейсов для~AML-аналитиков.


\section{Кейс QIWI: уроки по~vendor risk}
\label{sec:compliance-qiwi}

В~феврале 2024~года Банк России отозвал лицензию у~АО~\enquote{КИВИ~Банк}~\todocite{дата и~официальный пресс-релиз ЦБ об~отзыве лицензии у~QIWI Банка}.
Это~стало финалом длительной регуляторной истории: ЦБ неоднократно
выдавал предписания об~устранении нарушений в~части контроля транзакций
и~AML-процедур. История QIWI~--- показательный кейс не~только про~конкретный
банк, но~и~про~то, как~системные операционные риски могут накапливаться
незаметно.

Несколько наблюдений, которые извлекает индустрия из~этого кейса.
Прежде всего, регуляторные предписания~--- это~не~штрафы, а~предупреждения;
они~сигнализируют о~том, что~регулятор видит системную проблему, которую
оператор не~устранил, и~серия предписаний без~реакции~--- путь к~более
серьёзным последствиям. Далее, масштаб системы создаёт слепые
пятна: QIWI обрабатывала огромный объём микроплатежей~--- оплата
игр, переводы физлиц~--- и~транзакционный мониторинг этого потока
требовал ресурсов, которые не~всегда выделялись пропорционально росту.
Наконец, vendor~risk management~--- риск зависимости от~платёжного
провайдера~--- оказался реальным для~любого бизнеса, процессировавшего через QIWI:
при~отзыве лицензии средства клиентов заморожены, процессинг остановлен
в~один день.

\begin{important}
  Отзыв лицензии платёжного провайдера~--- событие, которое нельзя
  отладить в~понедельник. Для~бизнесов с~критической зависимостью от~одного
  провайдера~--- платёжный шлюз, кошелёк для~выплат, основной эквайер~---
  необходим план непрерывности: альтернативный провайдер в~горячем резерве,
  средства на~нескольких счетах. Это~не~паранойя, а~операционный стандарт.
\end{important}


\section{FATF в~СНГ-контексте}
\label{sec:compliance-fatf}

FATF~(Financial Action Task Force) издаёт 40~рекомендаций, которые
являются международным стандартом в~области противодействия отмыванию
денег и~финансированию терроризма~\sidecite{fatf-recs}. Страны имплементируют
эти~рекомендации в~национальное законодательство; FATF проводит взаимные
оценки и~публикует результаты, которые напрямую влияют на~доступ
к~корреспондентским банковским услугам.

Для~платёжного бизнеса в~СНГ особое значение имеют три аспекта. Первый~---
так называемый grey list: страны, получившие повышенный мониторинг FATF
(\enquote{серый список}), сталкиваются с~осложнёнными корреспондентскими
отношениями: западные банки применяют к~транзакциям из~таких стран
усиленный due diligence, что~повышает операционные затраты и~время
проведения платежа. Для~платёжного оператора в~стране из~серого
списка это~реальное конкурентное ограничение.

Второй аспект~--- эффективность реализации: FATF оценивает не~только
наличие законов, но~и~их~эффективное исполнение. Казахстан, Узбекистан,
Армения~--- страны с~развитыми национальными платёжными системами~---
имеют разный уровень соответствия FATF-рекомендациям. Белоруссия находилась
в~состоянии повышенного внимания FATF в~разные периоды~\sidecite{fatf-recs}\todocite{актуальный статус стран СНГ по~оценкам FATF, 2024--2025}.

Третий~--- де-рискинг: глобальные банки в~рамках управления
собственным compliance-риском могут полностью прекращать корреспондентские
отношения с~банками из~высокорисковых юрисдикций~--- явление, известное
как~de-risking. Для~платёжного оператора в~СНГ, зависящего от~западных
корреспондентов для~международных расчётов, это~означает необходимость
диверсификации корреспондентской сети.

\begin{practice}
  При~выходе в~новую юрисдикцию СНГ полезно сделать три проверки:
  актуальный FATF-статус страны (серый список или~нет), последняя взаимная
  оценка FATF (её~публичная часть описывает конкретные слабости AML-системы),
  и~наличие двусторонних соглашений об~обмене финансовой информацией
  с~Россией (влияет на~возможности мониторинга трансграничных операций).
  Всё~три источника~--- публичные и~не~требуют специального доступа.
\end{practice}


\section*{Итоги главы}

\begin{itemize}
  \item \law{115-ФЗ} обязывает платёжного оператора к~трём группам
        действий: идентификация клиентов (три уровня с~разными порогами),
        мониторинг операций и~направление сообщений в~Росфинмониторинг,
        заморозка при~совпадении с~перечнями.
  \item KYC физлиц строится на~уровнях идентификации (ЕСИА, паспорт,
        видеоверификация) с~нарастающими возможностями; ПЭП-статус
        требует Enhanced Due Diligence.
  \item KYB мерчантов объединяет проверку юридического лица, структуры
        собственности, UBO и~андеррайтинг по~MCC; PayFac-модель переносит
        ответственность на~платформу, не~снимая её.
  \item Санкционный скрининг должен охватывать четыре точки:
        онбординг, каждая транзакция, обновление данных, обновление
        списков; latency решается in-memory базами; false positive
        управляется эскалационным workflow.
  \item Кейс QIWI показывает: регуляторные предписания~--- предупреждения,
        а~не~штрафы; vendor risk management~--- стандарт, не~опция.
  \item FATF-статус страны влияет на~корреспондентские отношения:
        grey list~--- осложнённый де-рискинг; де-рискинг~--- потеря
        корреспондентов; актуальные оценки стран СНГ~--- публичны.
\end{itemize}
