\chapter{Глоссарий}

\begin{description}
  \item[\textbf{PAN}] Primary Account Number — основной номер карты; главный идентификатор карты в платёжной инфраструктуре.

  \item[\textbf{IIN}] Issuer Identification Number — первые 6 или 8 цифр PAN, идентифицирующие эмитента и карточную сеть.

  \item[\textbf{BIN}] Bank Identification Number — историческое название IIN, которое по-прежнему широко используется в индустрии.

  \item[\textbf{MII}] Major Industry Identifier — первая цифра PAN, указывающая на отрасль эмитента.

  \item[\textbf{корреспондентский счёт}] Счёт банка в другом банке или в Банке России, через который банк ведёт межбанковские расчёты.

  \item[\textbf{алгоритм Луна}] Алгоритм вычисления контрольной цифры PAN для обнаружения ошибок ввода; не является криптографической защитой.

  \item[\textbf{CVV}] Card Verification Value — код верификации, записанный на магнитной полосе карты и вычисляемый эмитентом.

  \item[\textbf{магнитная полоса}] Магнитный носитель на карте, который хранит статические данные и поэтому уязвим к копированию.

  \item[\textbf{ISO}] Международная организация по стандартизации; публикует отраслевые и технические стандарты.

  \item[\textbf{PCI DSS}] Стандарт безопасности данных индустрии платёжных карт, обязательный для участников карточной инфраструктуры.

  \item[\textbf{эмитент}] Банк или иная кредитная организация, выпускающая карту и принимающая решение по авторизации операции со стороны держателя.

  \item[\textbf{эквайер}] Банк или лицензированная организация, которая подключает мерчанта к приёму карт и зачисляет ему деньги по итогам расчёта.

  \item[\textbf{interchange}] Межбанковская комиссия, которую эквайер платит эмитенту по итогам карточной транзакции.

  \item[\textbf{PSP}] Payment Service Provider — провайдер платёжных услуг, который даёт мерчанту единый способ принимать платежи поверх одного или нескольких банковских партнёров.

  \item[\textbf{CNP}] Card-not-present — операция, в которой карта не предъявляется терминалу физически, а её реквизиты передаются удалённо.

  \item[\textbf{RBA}] Risk-Based Authentication — аутентификация, в которой решение о дополнительной проверке принимается по совокупности контекстных сигналов о транзакции и держателе.

  \item[\textbf{RSA}] Асимметричный криптографический алгоритм, который в платёжных системах часто используется для цифровых подписей и обмена ключами.

  \item[\textbf{JCB}] Международная карточная схема, возникшая в Японии и работающая на глобальном рынке платежей.

  \item[\textbf{SAD}] Sensitive Authentication Data — чувствительные аутентификационные данные, которые стандарт PCI DSS запрещает хранить после авторизации.

  \item[\textbf{hold}] Временная блокировка суммы после авторизации: деньги ещё не списаны окончательно, но уже недоступны для новых операций.

  \item[\textbf{reversal}] Сообщение отмены уже отправленной авторизации до клиринга; типичный способ быстро снять hold и убрать неоднозначность после сбоя.

  \item[\textbf{dual-message}] Модель карточной операции, в которой авторизация и финансовая запись разведены во времени: сначала сеть получает решение эмитента, затем — клиринговую запись.

  \item[\textbf{EMV}] EMV — стандарт микропроцессорных платёжных карт, обеспечивающий динамическую аутентификацию транзакций.

  \item[\textbf{EMVCo}] EMVCo — организация, которая поддерживает и развивает спецификации EMV.

  \item[\textbf{AID}] Application Identifier — уникальный идентификатор платёжного приложения на EMV-чипе.

  \item[\textbf{PSE}] Payment System Environment — каталог платёжных приложений на карте для контактного интерфейса.

  \item[\textbf{PPSE}] Proximity Payment System Environment — каталог платёжных приложений на карте для бесконтактного интерфейса.

  \item[\textbf{GPO}] Get Processing Options — команда EMV, с которой терминал запускает обработку выбранного приложения.

  \item[\textbf{AIP}] Application Interchange Profile — битовое поле, описывающее поддерживаемые функции EMV-приложения.

  \item[\textbf{AFL}] Application File Locator — список файлов и записей, которые терминал должен прочитать в ходе EMV-транзакции.

  \item[\textbf{SDA}] Static Data Authentication — базовый метод офлайн-аутентификации EMV-карты, проверяющий неизменность подписанных данных.

  \item[\textbf{DDA}] Dynamic Data Authentication — метод офлайн-аутентификации EMV, при котором карта формирует подпись на уникальных данных транзакции.

  \item[\textbf{CDA}] Combined DDA/Application Cryptogram Generation — EMV-метод, совмещающий аутентификацию карты с генерацией криптограммы транзакции.

  \item[\textbf{CVM}] Cardholder Verification Method — способ верификации держателя карты, например PIN, подпись или отсутствие проверки.

  \item[\textbf{ARQC}] Authorization Request Cryptogram — криптограмма EMV, означающая, что карта просит онлайн-авторизацию у эмитента.

  \item[\textbf{TC}] Transaction Certificate — криптограмма EMV, означающая офлайн-одобрение транзакции картой.

  \item[\textbf{AAC}] Application Authentication Cryptogram — криптограмма EMV, означающая офлайн-отклонение транзакции картой.

  \item[\textbf{TVR}] Terminal Verification Results — пятибайтовое EMV-поле, в котором терминал фиксирует результаты выполненных проверок.

  \item[\textbf{NFC}] Near Field Communication — радиоинтерфейс ближнего поля, который используется для бесконтактного обмена между картой или устройством и терминалом.

  \item[\textbf{CDCVM}] Consumer Device CVM — верификация держателя на самом устройстве, например биометрией или кодом разблокировки телефона.

  \item[\textbf{HCE}] Host Card Emulation — программная эмуляция платёжной карты на стороне операционной системы устройства.

  \item[\textbf{TSP}] Token Service Provider — сервис карточной сети или платёжной системы, который выпускает и обслуживает платёжные токены.

  \item[\textbf{DPAN}] Device PAN — токенизированный номер карты, привязанный к конкретному устройству или сценарию использования.

  \item[\textbf{3DES}] Triple Data Encryption Algorithm — унаследованный симметричный шифр, долго применявшийся в платёжной инфраструктуре для PIN и ключевых операций.

  \item[\textbf{AES}] Advanced Encryption Standard — современный блочный шифр, пришедший на смену DES в новых криптографических контурах.

  \item[\textbf{HSM}] Hardware Security Module — физически защищённый модуль, который хранит ключи и выполняет криптографические операции внутри собственного периметра.

  \item[\textbf{DUKPT}] Derived Unique Key Per Transaction — схема деривации, в которой для каждой транзакции получается уникальный производный ключ.

  \item[\textbf{PIN-блок}] Структурированное значение, в котором PIN подготавливают к передаче по сети без раскрытия его в открытом виде.

  \item[\textbf{CDE}] Cardholder Data Environment — среда данных держателя карты, то есть системы и сегменты сети, где хранится, обрабатывается или передаётся карточный номер и связанные с ним данные.

  \item[\textbf{SAQ}] Self-Assessment Questionnaire — анкета самооценки PCI SSC для организаций, подтверждающих соответствие упрощённым образом.

  \item[\textbf{BASE I / BASE II}] Принятые в VisaNet названия авторизационного и клирингового контуров сети.

  \item[\textbf{TAL}] Token Assurance Level — атрибут сетевого токена, отражающий силу проверки при его выпуске и пригодный для риск-оценки эмитентом.

  \item[\textbf{OCT}] Original Credit Transaction — исходная кредитовая транзакция карточной сети, с помощью которой деньги зачисляют получателю.

  \item[\textbf{AFT}] Account Funding Transaction — транзакция финансирования счёта, которой в push-платежах часто предваряют зачисление получателю.

  \item[\textbf{VAA}] Visa Advanced Authorization — сетевой риск-скоринг Visa, который дополняет решение эмитента в авторизационном контуре.

  \item[\textbf{GCMS}] Global Clearing Management System — клиринговая система Mastercard, принимающая и обрабатывающая сообщения IPM.

  \item[\textbf{DSRP}] Digital Secure Remote Payment — механизм Mastercard для удалённой оплаты с опорой на EMV-криптографию и токенизацию.

  \item[\textbf{ABU}] Automatic Billing Updater — сервис Mastercard для обновления сохранённых платёжных реквизитов после перевыпуска или истечения срока карты.

  \item[\textbf{MAC}] Merchant Advice Code — код рекомендации мерчанту в контуре Mastercard, который подсказывает, допустимы ли повторы и какие действия уместны после авторизационного ответа.

  \item[\textbf{ОПКЦ}] Операционный и платёжный клиринговый центр — инфраструктурный контур, который маршрутизирует карточные сообщения и сводит клиринговые позиции внутри платёжной системы.

  \item[\textbf{НКЦ}] Национальный клиринговый центр — расчётный центр, через который завершается денежный расчёт по клиринговым позициям участников.

  \item[\textbf{МРА}] Mir Payment Application — платёжное приложение карты «Мир», отвечающее за обработку транзакции на чипе и локальную криптографию.

  \item[\textbf{Mir Pay}] Мобильный платёжный сервис платёжной системы «Мир», который выпускает токен для устройства и позволяет платить телефоном в терминальном сценарии.

  \item[\textbf{UnionPay International}] Международный операционный контур UnionPay, через который сеть работает за пределами внутреннего китайского рынка.

  \item[\textbf{QuickPass}] Бесконтактный продукт UnionPay для оплаты картой, телефоном или носимым устройством.

  \item[\textbf{SBPay}] Мобильный сценарий оплаты через СБП, в котором приложение продавца передаёт платёжную ссылку в банковское приложение клиента.

  \item[\textbf{акцепт}] Заранее данное согласие клиента на последующие списания в пределах оговорённых условий по сумме, сроку и периодичности.

  \item[\textbf{A2A}] Account-to-account — перевод со счёта на счёт без карточного посредника.

  \item[\textbf{AISP}] Account Information Service Provider — лицензируемый участник открытого банкинга, который получает доступ к данным счёта и истории операций.

  \item[\textbf{PISP}] Payment Initiation Service Provider — лицензируемый участник, который инициирует платёж со счёта клиента в пользу мерчанта или другого получателя.

  \item[\textbf{ISO 20022}] Международный стандарт структуры финансовых сообщений, в котором сообщение описывается богатыми типизированными данными, а не только позицией поля.

  \item[\textbf{MTI}] Message Type Indicator — четырёхзначный код типа сообщения в ISO 8583, показывающий место сообщения в жизненном цикле операции.

  \item[\textbf{bitmap}] Битовая карта присутствия полей в ISO 8583, указывающая, какие элементы данных дальше есть в сообщении.

  \item[\textbf{STAN}] Systems Trace Audit Number — шестизначный номер трассировки, по которому процессинг связывает запрос и ответ и ищет транзакцию в журналах.

  \item[\textbf{БЭСП}] БЭСП — система банковских электронных срочных платежей, режим валового расчёта в реальном времени (RTGS) в платёжной системе Банка России для крупных и срочных межбанковских переводов.

  \item[\textbf{RTGS}] RTGS (Real-Time Gross Settlement) — режим валового расчёта в реальном времени, при котором каждый платёж обрабатывается индивидуально и окончательно без взаимозачёта.

  \item[\textbf{SWIFT}] SWIFT (Society for Worldwide Interbank Financial Telecommunication) — международная система передачи финансовых сообщений между банками и другими финансовыми организациями; сама по себе не является расчётной системой.

  \item[\textbf{СПФС}] СПФС — Система передачи финансовых сообщений Банка России, российская инфраструктура обмена финансовыми сообщениями, функционально близкая к SWIFT.

  \item[\textbf{оператор по переводу денежных средств}] Участник, который вправе принять распоряжение, списать деньги у плательщика и обеспечить их зачисление получателю.

  \item[\textbf{оператор платёжной системы}] Организация, которая устанавливает правила участия, порядок обмена сообщениями и модель расчётов для участников платёжной системы.

  \item[\textbf{мерчант}] Мерчант — торгово-сервисное предприятие или онлайн-продавец, принимающий карточные платежи через эквайера.

\end{description}
