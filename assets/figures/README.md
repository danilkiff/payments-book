# Соглашение об иллюстрациях

Исходник палитры и пример всех элементов: [`../inkscape-template.svg`](../inkscape-template.svg).
Файл открывается в Inkscape и используется как стартовая точка.

## Палитра

| Имя       | Hex       | Назначение                                          |
|-----------|-----------|-----------------------------------------------------|
| Ink       | `#1A2030` | Основной текст. Никогда не используется для линий.  |
| Muted     | `#5C647A` | Вторичный текст, нейтральные стрелки, тонкий каркас |
| Panel     | `#E8ECF7` | Заливка панели - заметный "лист" под группой блоков |
| Soft      | `#F5F7FD` | Фон-фон - едва различимая зона                      |
| AccentA   | `#4E63D9` | Нейтральный смысловой акцент №1                     |
| AccentB   | `#7759D6` | Нейтральный смысловой акцент №2                     |
| Good      | `#2F8B67` | Семантика "ok / разрешено / штатный путь"           |
| Warn      | `#B8821C` | Семантика "осторожно / условно / fallback"          |
| Bad       | `#C15462` | Семантика "нельзя / отказ / ошибка"                 |

Других цветов в готовых SVG быть не должно. Допустимо только `#ffffff` для негативного
пространства.

Задача AccentA и AccentB - **различать** две сущности на одной схеме (например,
"эмитент" и "эквайер"). Семантику **оценки** выполняет только триада Good/Warn/Bad.

Если на схеме нужно больше двух нейтральных акцентов - декомпозируем.

## Запрет: «школьная блок-схема»

Прямоугольники со скруглением, связанные стрелками в линейный или
разветвлённый flow, — школьная нотация. Такие фигуры не несут
информации сверх того, что прозa перечисляет нумерованным списком,
и не оправдывают полосы печатной книги.

Признаки школьной блок-схемы:

- 3–8 одинаковых блоков, в каждом — короткая подпись («Покупка»,
  «Авторизация», «Расчёт», «Отказ»).
- Стрелки соединяют блоки в одну цепочку или дерево решений с двумя-
  тремя ветвями типа «да/нет».
- Подпись блока совпадает или почти совпадает со словом, которое
  прозa использует в том же абзаце.

Что использовать вместо:

- **Sequence diagram** (swimlanes по горизонтали, время по вертикали) —
  если важно показать, кто кому передаёт сообщение и в каком порядке.
- **State machine** — если важны не шаги, а состояния и переходы;
  узлы — состояния, рёбра помечены событиями-триггерами.
- **Архитектурная топология** — если важно спатиальное отношение
  компонентов, а не временной порядок.
- **Decision tree** — если ветвление по условиям имеет смысл;
  у каждого ребра должно быть содержательное условие, не «да/нет».
- **Матрица / grid** — если сравниваешь N сущностей по M атрибутам.
- **Просто прозa** — если фигура просто иллюстрирует линейный
  нарратив, который уже идёт текстом. Удалить.

Если ни один из этих жанров не подходит, фигура не нужна.

## Заливки и обводки

Строго **Solid color + fill-opacity**, то есть `#2F8B67` с
`fill-opacity="0.15"`, а не `#b2f2bb`. Это даёт согласованную яркость
по всей книге и не зависит от того, насколько светлый монитор у иллюстратора.

- `fill-opacity="0.15"` - для всех цветов, кроме Warn.
- `fill-opacity="0.18"` - для Warn (жёлтый на белом фоне иначе пропадает).
- Stroke - полный, насыщенный цвет, без opacity.

Panel и Soft применяются как сплошная заливка без opacity: они и так
бледные.

Текст рисуется как `fill="#1A2030"` (Ink) или `fill="#5C647A"` (Muted)
без обводки. Нет смысла выставлять stroke у `<text>` - это даёт визуально
"жирный" шрифт на одном экране и ломаный на другом.

## Линии и стрелки

Толщины:

- `stroke-width="1"` - служебный каркас (рамка панели, сетка).
- `stroke-width="1.5"` - основные контуры блоков.
- `stroke-width="2"` - связи между блоками и стрелки.

Пунктир - `stroke-dasharray="6,3"`. Используется для асинхронных или
необязательных связей. Другие dash-паттерны не вводить.

Стрелки рисуются через `<marker>`-определения в `<defs>`. Шесть канонических
маркеров (плюс "-s"-варианты для marker-start):

`arr-AccentA  arr-AccentB  arr-Good  arr-Warn  arr-Bad  arr-Muted`

Имя маркера всегда соответствует семантическому имени цвета, не hex-коду.
Если в `defs` появился `arr-1e1e1e` или `arr-8b5cf6` - значит, фигура
ушла с template и её нужно вернуть.

Цвет stroke у линии и цвет marker должны совпадать. Иначе стрелка
"отрывается" от своей линии и читается как отдельная сущность.

## Шрифт

Семейство: `font-family="Inter, -apple-system, BlinkMacSystemFont, 'Helvetica Neue', Arial, sans-serif"`

Размеры:

- `font-size="11"` - заголовки секций внутри фигуры (`font-weight="bold"`).
- `font-size="10"` - основная подпись блока.
- `font-size="9"` - мелкие пометки, подписи стрелок.

Меньше 9 - нельзя. На печати такая подпись нечитаема.

## Inkscape workflow

- **Стартуем с шаблона.** `File / Open` на `assets/inkscape-template.svg`,
  `File / Save As` под новым именем в нужную папку главы. Не открывать
  "новый документ" - он принесёт настройки Inkscape, которые не
  совпадают с template (другой шрифт, другая сетка, другая обводка
  по умолчанию).

- **Не использовать Inkscape Color Picker** на пастельных образцах
  из тулбара или из чужих SVG. Из палитры Inkscape берутся `#b2f2bb`,
  `#a5d8ff` и подобные - это пастель Excalidraw, не наша палитра.
  Цвета вводим вручную через `Fill and Stroke / RGBA` и копируем
  hex из таблицы выше.

- **viewBox** - единственный источник истины для размеров. Не
  полагаться на `width`/`height` в физических единицах. Inkscape
  иногда добавляет `width="210mm" height="297mm"` - это безвредно,
  но сборка ориентируется на viewBox.

- **Шрифт через Text tool**, не "Convert to path". Текст должен
  остаться текстом - иначе он не масштабируется и не индексируется
  в PDF. Если шрифт у пользователя действительно не Inter - пусть
  fallback сделает своё дело, лишь бы текст оставался редактируемым.

- **Группировка** - по смыслу: один блок = одна группа. Inkscape
  создаёт `<g>` автоматически при выделении нескольких объектов и
  нажатии `Ctrl+G`. Бесполезные одиночные группы (один `<g>` с
  одним `<rect>` внутри) стоит развернуть `Ctrl+Shift+G`.

- **Маркеры стрелок.** Inkscape по умолчанию добавляет в `defs`
  маркер с автоматическим id вида `Arrow1Lend` и красит его в цвет
  stroke. Это не наш стиль. Заменить вручную в XML-редакторе
  (`Edit / XML Editor`) на один из шести `arr-*` маркеров,
  скопированных из template `defs`. Или, проще: сразу копировать
  готовую стрелку из template и менять только координаты.

- **Inkscape SVG output.** Сохранять как "Plain SVG" (`File / Save As / Plain SVG`).
  Это убирает Inkscape-специфичные атрибуты (`sodipodi:*`, `inkscape:*`),
  которые загромождают diff и не нужны для рендеринга.

## Проверка перед коммитом

Автоматизированно:

```bash
# 1. XML валиден
xmllint --noout assets/figures/chXX-name/figure.svg

# 2. В файле только template-цвета (плюс #ffffff)
grep -oE '#[0-9a-fA-F]{6}' assets/figures/chXX-name/figure.svg | sort -u
# ожидаем подмножество: #1A2030 #5C647A #E8ECF7 #F5F7FD
#                       #4E63D9 #7759D6 #2F8B67 #B8821C #C15462 #ffffff

# 3. Маркеры - только из канонических шести
grep -oE 'id="arr-[^"]*"' assets/figures/chXX-name/figure.svg | sort -u
# ожидаем подмножество arr-AccentA / arr-AccentB / arr-Good /
#                      arr-Warn / arr-Bad / arr-Muted (и их -s варианты)
```

После прохождения проверок `make svg` пересоберёт PDF из этого SVG
через Inkscape headless. Сборка PDF - обязательная часть коммита,
если SVG менялся: `.tex` подключает именно PDF, не SVG.

## Автоматизация: `scripts/figures/`

Фигуры с табличной/сеточной структурой генерируются Python-скриптами
из [`scripts/figures/`](../../scripts/figures/) -- это канон автоматизации
для фигур, где данные диктуют разметку. Скрипты пишут SVG напрямую,
используют общий модуль [`_common.py`](../../scripts/figures/_common.py)
с палитрой, шрифтом и helper'ами.

Текущие генераторы:

| Скрипт                          | Выход                                                  | Глава |
|---------------------------------|--------------------------------------------------------|-------|
| `gen_service_code.py`           | `ch04-card-data/service-code.svg`                      | 4     |
| `gen_mti_0100_anatomy.py`       | `ch06-iso8583/mti-0100-anatomy.svg`                    | 6     |
| `gen_bitmap_presence.py`        | `ch06-iso8583/bitmap-presence.svg`                     | 6     |
| `gen_pdol_anatomy.py`           | `ch07-emv/pdol-anatomy.svg`                            | 7     |
| `gen_tvr_anatomy.py`            | `ch07-emv/tvr-anatomy.svg`                             | 7     |
| `gen_tsi_anatomy.py`            | `ch07-emv/tsi-anatomy.svg`                             | 7     |
| `gen_aip_anatomy.py`            | `ch07-emv/aip-anatomy.svg`                             | 7     |
| `gen_pin_block_format0.py`      | `ch10-cryptography/pin-block-format0.svg`              | 10    |

Workflow:

```bash
# 1. Поправить данные в~скрипте (схема, пример, лейблы).
# 2. Перегенерировать SVG.
python3 scripts/figures/gen_<name>.py

# 3. Перегенерировать PDF из SVG.
make svg

# 4. Перегенерировать книгу.
make pdf
```

Соглашения для новых скриптов:

- Импортировать палитру и helpers из `_common.py`, не дублировать константы.
- Все пути -- через `figures_dir()` из `_common`; никаких абсолютных.
- Источники данных и формул цитировать в~docstring; внешние факты
  фиксировать в `assets/bibliography.bib` и ссылаться на ключи bib
  из подписи фигуры в~`.tex`.
- Inkscape ломает PDF export при наличии Unicode-операторов
  (например, `⊕`, `⊗`); использовать словесные эквиваленты
  (`XOR`, `AND`) внутри SVG-text.

Для фигур со свободной композицией (диаграммы потоков, схемы)
канон остаётся прежним: открыть `inkscape-template.svg`, нарисовать
вручную, сохранить как Plain SVG.
