# Канон терминов «Инженерии платежей»

Этот файл — единый источник истины для выбора терминов при написании и правке прозы книги. Сгенерирован из `build/glossary-work/clusters.py`. **Не редактировать вручную:** правки вносить в `clusters.py` и пересобрать через `python3 build/glossary-work/emit_canon.py`.

## Как читать

- `**канон**` — форма, которую следует использовать в прозе по умолчанию.
- `варианты: «X», «Y»` — другие формы того же термина. По умолчанию не используются; допустимы при первом упоминании или в техническом контексте, если иное не сказано в авторском правиле.
- _курсив_ под bullet'ом — авторское правило: когда канон строгий, когда вариант допустим, на что обращать внимание (омонимы, контексты).

Кластеры без вариантов (один-единственный канон без синонимов) опущены — там нечего выбирать.
